# Problem Statement
- 当前 `base` 模型在专有名词（如 Gemini、MiniMax）和长句断句上表现不稳定。
- 现有词表仅用于 Whisper prompt，约束力弱，无法稳定修正误识别（例如 `JMI`）。
- 模型选择被硬编码为 `ggml-base.bin`，无法切换到 `small` 进行精度提升。
- 非目标：默认启用远程 LLM（保持本地链路为主，远程仅可选增强）。

# Boundary & Ownership
- `src/main/index.js`: 模型选择、下载、运行时配置切换。
- `src/main/whisper-engine.js`: 转写参数与文本后处理入口。
- `src/main/config/config-manager.js`: 配置默认值与兼容迁移。
- `src/renderer/settings.html`: 模型与输出策略配置 UI。
- `src/main/model-downloader.js`: 模型元信息与下载提示。

# Options & Tradeoffs
1. 仅调 Whisper 参数（beam/best-of/max-len）
- 优点：实现快，无新依赖。
- 缺点：对专有名词误识别修复有限，难满足稳定纠错目标。拒绝单独采用。

2. 规则后处理 + small 模型（本次）
- 优点：低延迟、可离线、可解释；small 模型提升底层识别质量。
- 缺点：规则断句不是语义级最佳；需要维护词表替换映射。

3. 远程轻量 LLM 后处理（可选，后续扩展）
- 优点：语义断句与术语修正潜力更高。
- 缺点：网络依赖、成本与隐私风险、链路复杂。

4. 规则后处理 + 可选轻量 LLM（本次最终）
- 优点：默认低延迟离线，用户可按需开启联网增强；超时回退可控。
- 缺点：仍需维护参数（超时/长度阈值）与 API key。

# Migration Plan
1. 新增模型解析：`base/small` -> 模型文件名，取消硬编码 base。
2. 启动时按配置检查/下载模型；设置保存时支持运行时切换模型。
3. 新增词表后处理能力：支持 `vocabulary.json` 中显式替换映射。
4. 新增轻量中文断句/标点规则（可开关），默认开启。
5. 设置页增加 `model` 与 `punctuation` 相关项，默认 `base + 开启后处理`。
6. 保持向后兼容旧配置（字段缺失自动回填默认值）。
7. 增加可选 LLM 后处理配置（默认关闭，超时自动回退，长度阈值触发）。

# Test Strategy
- 单元/脚本验证：
  - 繁体转简体、术语替换、断句规则输入输出正确。
  - 旧配置文件缺失新字段时自动补默认值。
- 集成验证：
  - 首次启动选择 `small` 可下载并生效。
  - 设置中切换 `base/small` 后无需重启可继续识别。
  - 长语音输入观察断句与术语稳定性提升。
- 验收标准：
  - `Gemini/MiniMax/Kimi` 等词可通过后处理稳定成目标写法。
  - 长句至少具备基础断句（逗号/句号）而非整段无标点。
  - 启用 LLM 后处理时，异常/超时不影响主流程输出（可回退到本地结果）。
